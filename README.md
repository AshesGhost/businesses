## Engagement Process

| # | Phase | Description |
|---|-------|-------------|
| 01 | **Initial Contact & Scoping** | Define engagement scope, targets, rules of engagement, and sign an NDA / authorization letter. No surprises. |
| 02 | **Reconnaissance** | Passive and active information gathering — DNS enumeration, OSINT, port scanning, service fingerprinting — mapping the attack surface. |
| 03 | **Exploitation & Testing** | Controlled attempts to exploit discovered vulnerabilities: web attacks (SQLi, XSS, IDOR, misconfigs), network pivoting, privilege escalation. |
| 04 | **Written Report Delivery** | A structured PDF report with risk ratings, evidence screenshots, reproduction steps, and prioritized remediation advice. |
| 05 | **Re-Test (FREE)** | After fixes are applied, I verify the vulnerabilities are properly closed. All current engagements include a free re-test. |

## Scope & Ethics

**In scope:**
- ✅ Systems you own or have **explicit written authorization** to test

- ✅ CTF challenges, home labs, your own VPS / server

**Out of scope:**
- ❌ Unauthorized testing of third-party infrastructure — not now, not ever
- ❌ Offensive operations with no legitimate defensive purpose
